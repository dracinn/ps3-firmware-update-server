# Nexus PS3 Updater PS3 Components

This directory contains the PS3-side companion for PS3 Firmware Update Server:

- `ps3-update-helper.pkg`: installable status app that checks whether the desktop server is reachable and reports the compatible firmware selection currently served by the Mac app.

Neither component flashes firmware directly. Firmware installation still happens through:

```text
Settings > System Update > Update via Internet
```

## What They Check

Both components read these desktop app endpoints:

```text
http://<mac-ip>/api/status
http://<mac-ip>/api/firmware/manifest.json
```

The manifest is generated by the Kotlin app from the currently selected system profile, hardware status, main variant, and child variant. Firmware shown in `compatibleFirmware` has already passed the app's compatibility filter.

The status app writes its latest check to:

```text
/dev_hdd0/tmp/nexus-update-helper-status.txt
```

The app displays this file when it exists, then performs a fresh direct server check.

## GitHub Actions Build

This repository can build the status app with [FirebirdTA01/PS3DK](https://github.com/FirebirdTA01/PS3DK) in GitHub Actions.

Open **Actions > Build PS3 Helper > Run workflow**, then enter the Mac IP address that the PS3 should contact. The workflow uploads:

```text
ps3-update-helper.elf
ps3-update-helper.self
ps3-update-helper.fake.self
ps3-update-helper.pkg
ps3-update-helper.gnpdrm.pkg
SHA256SUMS.txt
```

The first run builds and caches PS3DK, so it can take a while. Later runs should reuse the cached toolchain.

## Local VM Build

On macOS, this repository can build the helper inside an Ubuntu Lima VM:

```bash
brew install lima
./ci/vm/start-ps3-helper-vm.sh
./ci/vm/build-ps3-helper-in-vm.sh 192.168.1.50
```

VM-built artifacts are copied to:

```text
artifacts/ps3-update-helper-vm/
```

## PS3 Install

Copy `ps3-update-helper.pkg` or `ps3-update-helper.gnpdrm.pkg` to `/dev_hdd0/packages/`, then install it from Package Manager. Run it whenever you want an on-screen status check from the console.

## Local Build Requirements

- PSL1GHT, PS3DK, or another PS3 homebrew SDK with `ppu-gcc`
- Network access from the PS3 to the Mac running the desktop app

Build with a default server IP baked in:

```bash
make SERVER_IP=192.168.1.50
```

Clean build artifacts:

```bash
make clean
```

The Makefile produces `build/ps3-update-helper.elf` for quick status app compile checks. Use the CMake or VM build for packaged app artifacts.

When using PS3DK locally, you can also build with CMake:

```bash
source /path/to/PS3DK/scripts/env.sh
cmake -S . -B cmake-build \
  -G Ninja \
  -DCMAKE_TOOLCHAIN_FILE="$PS3_TOOLCHAIN_ROOT/cmake/ps3-ppu-toolchain.cmake" \
  -DSERVER_IP=192.168.1.50
cmake --build cmake-build
```

## Workflow

1. Open PS3 Firmware Update Server on the Mac.
2. Select the PS3 system profile and hardware status.
3. Select remote or local firmware and make sure a compatible `PS3UPDAT.PUP` is ready.
4. Start the server.
5. Copy and install `ps3-update-helper.pkg` on the PS3.
6. Run the status app on the PS3 and confirm the server is reachable.
7. Set the PS3 DNS to the Mac IP shown in the desktop app.
8. Install through `Settings > System Update > Update via Internet`.

If the helper cannot reach the desktop app, check that the Mac and PS3 are on the same network and that macOS allows incoming connections on HTTP port 80.
